  Why It Is Not Exactly Matching The Original                                                                                                                                                              
  The main reasons are implementation-path and model-path differences, not a broken method.                                                                                                                
                                                                                                                                                                                                           
  - The benchmark reproduction retrains fresh benchmark MLPs from seeds [42, 43] in experiment/cemsp/config.yml:43, while the original repo Hepatitis script uses two existing checkpoints in reference/   
    CEMSP/cfgen-update/hepatitis/cemsp.py:33. That alone can materially change the returned CF sets and the retraining inconsistency.                                                                      
  - The benchmark model code is not the same training/inference code as the original repo’s NNModel/checkpoint flow. The architecture and hyperparameters were aligned, but exact weights are not expected 
    to match bit-for-bit.                                                                                                                                                                                  
  - The current run used cuda because it was available, while the original Hepatitis training script in reference/CEMSP/train/hepatitis/train_hepatitis.py:116 used CPU. That can change optimization      
    trajectories slightly even with fixed seeds.                                                                                                                                                           
  - The benchmark implementation had to preserve the repo’s MethodObject API in method/cemsp/cemsp.py, so it is a faithful reimplementation inside this framework, not a literal reuse of the original     
    source file from reference/.                                                                                                                                                                           
  - The largest remaining gap is on the second-model CF set behavior, which shows up in mean_diversity2 and especially mean_inconsistency. That is consistent with the model-pair not matching the original
    checkpoint pair exactly.                                                                                                                                                                               
  - The normal-range setup itself is aligned with the original Hepatitis script at reference/CEMSP/cfgen-update/hepatitis/cemsp.py:50, so the residual mismatch is not coming from the bounds.
